Event Filters

Sets of filters used to filter specific events.

Examples of event filters:

script.set_event_filter(defines.events.on_built_entity, {{filter = "name", name = "fast-inserter"}})
script.set_event_filter(defines.events.on_entity_died, {{filter = "type", type = "unit"}, {filter = "name", name = "fast-inserter"}})

script.on_event(defines.events.on_entity_damaged,
  function(e) game.print("A wall was damaged on tick " .. e.tick) end,
  {{filter="type", type = "wall"}})

LuaScriptRaisedReviveEventFilters

An array of filters. Each filter is a table:

  • filter :: string: The condition to filter on.
  • mode :: string (optional): How to combine this with the previous filter. Must be "or" or "and". Default is "or".
  • invert :: boolean (optional): Inverts the condition. Default is false.

Depending on the value of filter, the table may take additional fields. filter may be one of the following:

  • "ghost"
  • "rail"
  • "rail-signal"
  • "rolling-stock"
  • "robot-with-logistics-interface"
  • "vehicle"
  • "turret"
  • "crafting-machine"
  • "wall-connectable"
  • "transport-belt-connectable"
  • "circuit-network-connectable"
  • "type": Additional fields:
    • type :: string: The prototype type
  • "name": Additional fields:
    • name :: string: The prototype name
  • "ghost_type": Additional fields:
    • type :: string: The ghost prototype type
  • "ghost_name": Additional fields:
    • name :: string: The ghost prototype name
