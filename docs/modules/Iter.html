deep_pairs(tbl, opt)
A for-loop compatible depth-first non-recursive STATEFUL iterator. The iterator does not take any input and always returns the next items. Key finding uses next(). When a loop is encoutered with verboselooplogging != nil then any later found entrance points to the loop will be ignored.

LUA MANUAL about next(): The behavior of next is undefined if, during the traversal, you assign any value to a non-existent field in the table. You may however modify existing fields. In particular, you may clear existing fields.

Parameters:

  • tbl table the table to be iterated.
  • opt deep_pair_options or nil

Tables

deep_pair_options

Fields:

  • max_depth int or nil 10000, paranoia option to prevent freezing on very large objects
  • verbose_loop_logging boolean or nil nil:error,true:logging,false:silent
  • include_subtables_in_output boolean or nil false
  • return_path boolean or nil false (include copy of path table in output? slower.)
